# Resume Website

Сайт-визитка с поддержкой многоязычности (русский/английский), построенный на Node.js, TypeScript и Handlebars. Генерирует статические HTML файлы.

## Технологии

- Node.js
- TypeScript
- Handlebars
- Chokidar (для watch режима)

## Установка

```bash
npm install
```

## Сборка

### Одноразовая сборка

```bash
npm run build
```

Сгенерирует статические HTML файлы в папку `dist/`:
- `dist/index.html` - русская версия
- `dist/index-en.html` - английская версия
- `dist/styles.css` - стили

### Режим разработки (watch)

```bash
npm run dev
```

Автоматически пересобирает сайт при изменении:
- Шаблонов Handlebars (`views/**/*.hbs`)
- Файлов локалей (`locales/**/*.json`)
- Стилей (`styles.css`)

## Структура проекта

```
resume/
├── src/
│   └── build.ts           # Скрипт сборки статических файлов
├── views/

│   └── index.hbs          # Главная страница

│   └── en.json            # Английская локализация

├── styles.css             # Стили (BEM методология)

├── tsconfig.json
└── README.md
```

## Деплой на GitHub Pages

Проект настроен для автоматического деплоя на GitHub Pages через GitHub Actions.

### Настройка GitHub Pages

1. Перейдите в настройки репозитория: `Settings` → `Pages`
2. В разделе `Source` выберите `GitHub Actions`
3. Сохраните изменения

### Автоматический деплой

После настройки, каждый push в ветку `main` или `master` автоматически:
- Соберет проект
- Задеплоит содержимое папки `dist/` на GitHub Pages

Workflow файл находится в `.github/workflows/deploy.yml`

### Альтернативные хостинги

После сборки (`npm run build`) папка `dist/` содержит готовые статические файлы, которые можно загрузить на любой хостинг:

- Netlify
- Vercel
- Любой статический хостинг

Просто загрузите содержимое папки `dist/` на хостинг.
